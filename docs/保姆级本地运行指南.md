# 🚀 保姆级本地运行指南

本文档手把手教你在本地跑起来 AI 万能视频下载总结器的完整前后端，**零基础也能跟着做**。



## 前置要求

| 工具 | 版本要求 | 检查命令 | 安装方式 |
|------|----------|----------|----------|
| Python | ≥ 3.10 | `python3 --version` | [官网下载](https://www.python.org/downloads/) |
| Node.js | ≥ 18 | `node -v` | [官网下载](https://nodejs.org/en)（推荐 LTS 版本） |
| npm | ≥ 9 | `npm -v` | 随 Node.js 一起安装 |
| ffmpeg | 任意 | `ffmpeg -version` | [官网下载](https://ffmpeg.org/download.html) 或 `brew install ffmpeg`（macOS） |
| Git | 任意 | `git --version` | [官网下载](https://git-scm.com/) |

💡 关于 ffmpeg：ffmpeg 用于合并视频和音频流（如 YouTube 高清视频的画面和声音是分开的）。如果不安装，项目仍可运行，但下载高清视频时会降级为低画质单一格式。**强烈建议安装。**



## 第一步：克隆项目

```bash
git clone https://github.com/liyupi/free-video-downloader.git
cd free-video-downloader
```

💡 如果 GitHub 访问较慢，可以在 Gitee 上导入仓库后克隆。



## 第二步：获取 API Key

项目需要 **1 个必需的 API Key**，另外 1 组为可选。



### 必需：DeepSeek API Key

DeepSeek 用于 AI 视频总结功能（字幕提取后调用大模型生成总结、思维导图、问答）。

1. 打开 [DeepSeek 开放平台](https://platform.deepseek.com/)，注册并登录
2. 进入「API Keys」页面，点击「创建 API Key」
3. 复制生成的 Key（以 `sk-` 开头），妥善保存

💡 新注册用户通常会赠送免费调用额度，足够学习使用。

![](https://pic.yupi.icu/1/1772534666069-27ae688b-d917-4341-ae42-a3dbb5b26b89.png)



### 可选：Stripe 支付配置

Stripe 用于 VIP 会员的在线支付功能。**本地学习时可以不配置**，不影响视频下载和 AI 总结等核心功能。

如果你想体验完整的支付流程：

1）访问 [Stripe Dashboard](https://dashboard.stripe.com/)，注册并登录

2）确保处于「测试模式」（Test mode）

![](https://pic.yupi.icu/1/1773828514541-4e997b12-9f0e-4e4a-8fd8-579a3ca0f3fa-20260323193408848.png)

3）在「开发者 → API 密钥」获取 Secret Key（以 `sk_test_` 开头）

4）在「产品」页面创建一个产品和价格，获取 Price ID（以 `price_` 开头）

![](https://pic.yupi.icu/1/1773833187544-ca540fca-38ca-4225-a7aa-f9f502e0dcfc.png)

5）安装 [Stripe CLI](https://docs.stripe.com/stripe-cli)，先执行 `stripe login` 登录：

![](https://pic.yupi.icu/1/1773829060696-1563dc00-3483-498c-a229-bf7c87e1308d.png)

然后运行以下命令启动本地 Webhook 转发：

```bash
stripe listen --forward-to localhost:8000/api/payment/webhook
```
终端会输出 Webhook Signing Secret（以 `whsec_` 开头）

![](https://pic.yupi.icu/1/1773831388188-6f926d9a-c9c6-4a38-a5d4-8f56fac72fce-20260323193106527.png)



## 第三步：配置环境变量

```bash
cd backend
cp .env.example .env
```

用任意文本编辑器打开 `backend/.env` 文件，填入你的配置：

```env
# ✅ 必填 - DeepSeek API Key，用于 AI 视频总结
DEEPSEEK_API_KEY=sk-你的真实key

# ✅ 必填 - JWT 密钥，用于用户登录认证
# 本地开发可以随便写一个字符串，生产环境请使用强随机值
JWT_SECRET=my-local-dev-secret-123

# 🔧 选填 - Stripe 支付配置（不配置不影响核心功能）
STRIPE_SECRET_KEY=sk_test_你的stripe密钥
STRIPE_WEBHOOK_SECRET=whsec_你的webhook密钥
STRIPE_PRICE_ID_MONTHLY=price_你的价格id

# 🔧 选填 - 前端 URL，Stripe 支付完成后跳转地址
FRONTEND_URL=http://localhost:5173
```

| 字段 | 必填 | 用途 |
|------|------|------|
| `DEEPSEEK_API_KEY` | ✅ 必填 | 调用 DeepSeek 大模型生成视频总结、思维导图、AI 问答 |
| `JWT_SECRET` | ✅ 必填 | 用户登录 Token 的签名密钥，本地开发可随意设置 |
| `STRIPE_SECRET_KEY` | 🔧 选填 | Stripe 支付 API 密钥，不填则支付功能不可用 |
| `STRIPE_WEBHOOK_SECRET` | 🔧 选填 | Stripe Webhook 签名验证密钥 |
| `STRIPE_PRICE_ID_MONTHLY` | 🔧 选填 | Stripe 月度会员的价格 ID |
| `FRONTEND_URL` | 🔧 选填 | 支付完成后跳转地址，默认 `http://localhost:5173` |

⚠️ **注意**：`.env` 文件包含敏感信息，已被 `.gitignore` 排除，不会提交到代码仓库。



## 第四步：安装依赖

### 后端依赖（Python）

```bash
cd backend
pip install -r requirements.txt
```

💡 如果下载慢，可以切换为国内镜像：

```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```



### 前端依赖（Node.js）

```bash
cd ../frontend
npm install
```

💡 如果 npm install 速度慢，可以先切换为国内镜像：
```bash
npm config set registry https://registry.npmmirror.com
```



## 第五步：初始化数据库

本项目使用 **SQLite**，**无需手动安装任何数据库软件**。

后端服务启动时会自动完成以下操作：
1. 在 `backend/data/` 目录下创建 `app.db` 数据库文件
2. 自动建表（`users` 用户表、`orders` 订单表）

💡 如果需要重置数据库，直接删除 `backend/data/app.db` 文件，重启后端即可自动重建。

## 常见问题

### Q1：`ffmpeg` 未安装，下载视频报错或画质很低？

**原因**：高清视频的画面和声音是分开的，需要 ffmpeg 合并。

**解决**：
```bash
# macOS
brew install ffmpeg

# Ubuntu / Debian
sudo apt install ffmpeg

# Windows：从 https://ffmpeg.org/download.html 下载，解压后将 bin 目录加入 PATH
```

### Q2：`yt-dlp` 版本过低，解析视频失败？

**现象**：部分视频解析失败，报错 `Unable to extract` 或 `Unsupported URL`。

**解决**：
```bash
pip install --upgrade yt-dlp
```

### Q3：后端启动报环境变量未设置？

**原因**：`.env` 文件未创建或 Key 未填写。

**解决**：确认 `backend/.env` 文件存在，且 `DEEPSEEK_API_KEY` 已填入有效的 Key。

### Q4：端口 8000 或 5173 被占用怎么办？

**解决**：
```bash
# 查看占用端口的进程
lsof -i :8000    # macOS / Linux

# 终止占用的进程
kill -9 <PID>
```

### Q5：前端请求后端接口返回 CORS 错误？

**解决**：确保通过 `http://localhost:5173` 访问前端页面。前端会自动将 `/api` 请求通过 Vite 代理转发到后端，本项目后端也已配置了 `allow_origins=["*"]`。

### Q6：下载抖音视频失败？

**解决**：
1. 确保使用的是抖音分享链接（如 `https://v.douyin.com/xxx`）
2. 检查网络是否正常，抖音接口可能有频率限制
3. 尝试升级 yt-dlp：`pip install --upgrade yt-dlp`

### Q7：Stripe 支付功能测试不成功？

**解决**：
1. 确保 `.env` 中三个 Stripe 配置项都已填写
2. 确保 Stripe Dashboard 处于「测试模式」
3. 测试支付时使用 Stripe 提供的测试卡号：`4242 4242 4242 4242`
4. 本地测试 Webhook 需运行 Stripe CLI：
   ```bash
   stripe listen --forward-to localhost:8000/api/payment/webhook
   ```

### Q8：`pip install` 安装依赖报错？

**常见情况**：
- **bcrypt 编译失败**：安装 C 编译工具。macOS 运行 `xcode-select --install`
- **Python 版本过低**：需要 Python ≥ 3.10
- **网络超时**：使用国内镜像 `-i https://pypi.tuna.tsinghua.edu.cn/simple`

### Q9：Windows 系统运行终端命令报错？

**解决**：建议在 VSCode 中把默认终端改为 Git Bash，或使用 PowerShell。避免使用 CMD。



## 停止项目

在各终端窗口按 `Ctrl + C` 即可停止前后端服务。

数据库文件（`backend/data/app.db`）会保留，下次启动时数据不会丢失。
